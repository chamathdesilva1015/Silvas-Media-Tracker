SILVA'S MEDIA TRACKER: 10 PREMIUM UI & DESIGN EVOLUTIONS
======================================================

1. ADAPTIVE GLASSMORPHISM & DYNAMIC BLOB THEMING [DONE]
   - PC: Enhance the background "mesh-blobs" to react to the current category. For Anime, use vibrant neon gradients; for Movies, use deep cinematic blues and golds. The sidebar and cards should use a 15% opacity blur (glassmorphism) that reveals these blobs as they move.
   - Mobile: Simplify blobs to a single, slow-moving glow at the bottom of the viewport to keep the UI snappy.
   - Design Goal: Make the app feel alive and "organic" rather than a static grid of boxes.

2. IMMERSIVE HERO TRANSITIONS (MODAL 2.0) [DONE]
   - PC: When clicking an entry, the card shouldn't just "open" a modal. It should expand visually (Hero Animation), with the poster sliding into position and metadata fading in around it. The background should deeply dim with a heavy blur.
   - Mobile: Use a "Bottom Sheet" design. The profile should slide up from the bottom, covering 90% of the screen, allowing users to swipe it back down to close.
   - Design Goal: Create a sense of physical space and hierarchy, making the "Profile" feel like a destination.


3. HAPTIC-DRIVEN MOBILE GESTURES
   - Mobile: Implement "Swipe to Action" on the main grid. Swiping a card right triggers a "Like" (with a haptic "pop"), and swiping left opens the quick-edit menu. Long-pressing a card should trigger a subtle scale-down and a haptic pulse before showing a context menu.
   - PC: Replace swipes with "Hover Actions"—mini buttons that appear on the card corners (Heart, Edit, Delete) only when the mouse is over them.
   - Design Goal: Reduce the number of taps/clicks required for common tasks while making the interface feel premium and tactile.

4. VISUAL "VIBE" FILTERING (MOOD-BASED UI)
   - Concept: Instead of just "Genre: Horror," add a visual slider or icon set for "Vibe."
   - Detail: Use icons for "Chilled," "Heart-pounding," "Brain-bending," or "Visually Stunning." Each vibe change could subtly shift the accent color of the site (e.g., "Brain-bending" turns the site deep purple/violet).
   - Design Goal: Help users choose what to watch based on their current emotional state, not just technical metadata.

5. CINEMATIC FULL-SCREEN BACKDROP (PC ONLY) [DONE]
   - Concept: When viewing a specific entry's profile, the entire site background should fetch a high-res "backdrop" (fanart/still) from TMDB/Jikan.
   - Detail: Use a "Vignette" overlay (dark edges) to ensure the text remains perfectly readable. This turns the profile view into a mini-theatrical experience.
   - Design Goal: Maximize the visual impact of the media the user is currently interested in.


6. MICRO-INTERACTION FEEDBACK SYSTEM
   - Detail: Add "celebration" micro-animations. When a user marks something as 10/10 or "Liked," trigger a subtle burst of golden particles or a soft glow around the card. When "Passing" a suggestion, have the card "slide out" with a slight tilt as if being discarded.
   - Design Goal: Provide "dopamine hits" for data entry, making the process of tracking media feel rewarding and fun.

7. UNIVERSAL GRID LAYOUT TOGGLES
   - Concept: Let users choose their "Density."
   - Detail: Provide three icons in the header:
     * "Cinematic": Large posters, few details (Great for browsing).
     * "Standard": Current view.
     * "Data-Rich": Small posters, visible scores, genres, and directors in the grid (Great for power users).
   - Mobile: Default to "Standard" but allow a "Compact" list-view for scrolling through long libraries.
   - Design Goal: Respect the user's browsing style—some want the art, some want the data.

8. FLOATING GESTURAL NAVIGATION (MOBILE) [DONE]
   - Mobile: Move the main category navigation (Movies, TV, Anime, Manga) to a "Floating Tab Bar" at the bottom of the screen, easily reachable with the thumb. 
   - Detail: The bar should hide when scrolling down and reappear when scrolling up (Active Scroll logic).
   - Design Goal: Solve "Reachability" issues on modern tall smartphones, making one-handed use effortless.


9. VARIABLE TYPOGRAPHY & HIERARCHY [DONE]
   - Detail: Switch to a more premium font pair like 'Outfit' for headings and 'Inter' for body text. Use variable font weights (300 to 800) to create a clear hierarchy. For example, the Rating should be bold and large, while the "Directed by" text should be light and uppercase for a "technical" feel.
   - Design Goal: Improve readability and give the site a "high-end editorial" look similar to Apple TV or Criterion Channel.


10. RICH SKELETON SHIMMERS & LOADING STATES [DONE]
    - Detail: Replace generic "Loading..." text with Skeleton Screens (gray shapes that mimic the card layout) with a CSS "shimmer" animation.
    - Concept: For the Suggestion Hub, use a "radar" animation while the engine "thinks," making the wait feel like an intentional part of the experience.
    - Design Goal: Reduce perceived wait times and make the site feel polished even when the network is slow.

11. MANDATORY RATINGS & DATA INTEGRITY [DONE]
    - Detail: Enforce numeric ratings at both the database and UI levels to prevent "vanishing ratings" and ensure a consistent library.
    - Design Goal: Protect the user's data and ensure all metrics (averages, rankings) are always accurate.
